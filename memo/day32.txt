JSX(JavaScript XML)
	JavaScript에 XML(HTML) 개념 및 문법을 추가한 JavaScript 개발 문법이다.
	공식적인 개발 방법은 아니며, 리액트에서만 사용되는 개발 문법이다.

JSX 주의 사항
	1. 반드시 부모 요소 하나가 감싸는 형태여야 한다.
		부모가 2개 이상이면 오류이다.
		반드시 부모는 1개로 만든다.

	2. 자바스크립트 표현식
		JSX 안에서도 자바스크립트 표현식을 {}을 감싸 주어야 사용할 수 있다.

	3. if문(for문) 대신 삼항연산자(조건부 연산자) 사용
		if구문과 for 루프는 JavaScript 표현식이 아닌 문법이기에 사용할 수 없다.

	4. JSX 내에서 주석 사용 방법
		{/* 주석은 이렇게 ..*/} 

컴포넌트(Component)
	리액트에서 컴포넌트는 Props와 State를 가지는 Object다.
	컴포넌트는 props를 입력으로 받고 컴포넌트에서 엘리먼트를 조합하여
	렌터링을 위한 DOM을 출력으로 반환하는 객체이다.
	즉, props를 입력으로 받아 DOM을 출력으로 반환한다.


리액트 엘리먼트
	React 엘리먼트는 <div>, <span>과 같은 하나의 DOM에 해당하는 요소를 의미한다.

따라서 리액트 컴포넌트는 이 엘리먼트들이 모인 구조가 하나의 컴포넌트이다.
엘리먼트들을 모아 모듈화한 재사용 가능한 크기의 집합을 컴포넌트라고 할 수 있다.

컴포넌트 종류
	클래스형 컴포넌트 	리액트 초기의 컴포넌트 방식(구식)
	함수형 컴포넌트		리액트가 업데이트되며 만들어진 간결한 방식(신식)


컴포넌트 생명주기(LifeCycle)
	컴포넌트가 생성/업데이트/제거될 때의 컴포넌트 동작 과정을 의미한다.
	
	생성(mount)
		컴포넌트의 인스턴스가(객체)가 생성되어 DOM에 삽입되는 과정
	업데이트(updat)
		props 또는 state가 변경되어 렌더(화면그리기)가 진행되는 과정
	제거(unmount)
		컴포넌트가 DOM에서 제거되는 과정

생성 주기
	consturctor()
		컴포넌트가 생성될 때 단 한번만 호출되며, this.props, this.state에 접근할 수 있다.

	getDerivedStateFromProps()
		props에 있는 값을 state에 동기화 시킬 때 사용하는 메소드(선택사항)
		리액트 v16.3 이후에 새로 만든 라이프 사이클 메소드

	render()
		웹 화면(UI)를 렌더링(화면 그리기)하기 위해 DOM을 반환하는 메소드

	componentDidMount()
		컴포넌트가 웹 브라우저 상에 렌더링(그려진) 후 호출되는 메소드

업데이트 주기
	getDerivedStateFromProps()
		생성 주기에서 호출되 듯 업데이트가 시작되기 전 호출된다.

	shouldComponentUpdate()
		props/state를 변경했을 때 리렌더링을 시작할지 여부를 지정할 수 있다.
		true 이후에 업데이트 주기 진행, false 이후에 업데이트 주기 종료

	render()
		생성 주기와 동일하게 호출된다.

	getSnapshotBeforeUpdate()
		컴포넌트 변화를 DOM에 반영하기 바로 직전에 호출하는 메소드

	componentDidUpdate()
		컴포넌트 업데이트 작업이 끝난 후 호출하는 메소드

제거 주기
	componetWillUnmount()
		컴포넌트가 DOM에서 제거되기 전에 호출되는 메소드
