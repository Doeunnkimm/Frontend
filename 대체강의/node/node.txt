쿠키와 세션

쿠키
    쿠키는 브라우저에 저장되는 키와 값이 들어있는 작은 데이터 파일
    하나당 4kb만 저장, 최대 300개

    사용자가 유효기간을 명시, 유효기간이 정해지면 브라우저가 종료되어도
    쿠키는 유지, 이 점을 이용하여 자동 로그인 기능이 구현

    설정되어 있지 않다면 브라우저를 닫는 순간 삭제
    
    ex)
        자동로그인
        팝업창 3일 동안 보지 않기

    
세션
    세션은 쿠키를 기반하여 세션 ID를 사용자에게 전달
    서버측에 사용자 구분을 위해 사용 따라서 보안 해야할 필요가 있는 데이터를 서버에서 관리하기 용이

    그러나, 사용자가 많아질 수록 서버에서 가지고 있는 세션의 갯수 증가
    운영하는 측 = 데이터 비용 부담이 증가

    redis(key-value DB)를 통해 관리하는 경우가 많음
    로그인, 민증정보와 같은 보안상 중요한 작업을 수행할 때 사용

ORM (Object Relational Mapping), 객체와 관계형 DB를 연결

    TypeORM + nextJS ( oop를 곁들인 js )
    Sequelize + express 

    * Sequelize
     - ORM의 한 종류, SQL문을 사용하지 않아도 명령문 만으로 DB 관리가 가능
       필요에 따라 Row Query(SQL 쿼리문)도 사용 가능

    npm i mysql2, sequelize, sequelize-clis
    npx sequelize init

    config
    DB와 연결 설정

    models
    테이블 정의 및 스키마 구성

    migrations
    테이블, 스키마 수정

    seeders
    백업본 저장
